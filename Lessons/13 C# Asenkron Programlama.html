Asenkron Programalama

Asenkron programalama, aynı anda iki veya ikiden fazla işi yapabilen ve yürütebilen programlamaya denir.
Programlar genelde iki türle yazılır. Senkron ve asenkron. Senkron programlar, tek bir Thread çalışırlar; Asenkron ise birden fazla Thread ile aynı zamanda birden fazla işi yaparlar. Performans bakımından asenkron programlar, işlemcinin çekirdek sayısına bağlı olarak senkronlardan daha performanslıdırlar.
Thread'leri aslında birer işçiye benzetebiliriz. Bu dijital işçi bizim verdiğimiz görevleri sırayla yapan bir yapıdadır. Senkron programlar tek bir işçisi vardır. Bu sebeble aynı anda iki işi yapamazlar. İşte bu durumda asenkron programlama devereye girerek, yeni dijital işçi alarak yani yeni Thread açarak, bazı işleri diğer işçilere vermemizi sağlar. Kullanıldığı durumlarda genelde Web teknolojileri, low-level programlama ve yeni yeni popüler olan AI programlamada kullanılır. Böylelikle aynı zamanda birden fazla iş yapılır.
Başlangıç yani Main fonksiyonu olduğu gibi de Main Thread vardır. Bu Thread, programın başladığı Thread'dır. Aynı zamanda önceki sayfalarda bahsettiğim program akışı, Thread'lerden bahsediyordum ki Thread'lerde de aynı mantık vardır. Aynı anda iki işi yapabilmek için de ikiden fazla Thread yani program akışı gereklidir.

Task

Task'ler, bizim için o dijital işçilermizdir. Basit bir şekilde birden fazla işi yapacaksak eğer Task kullanıyoruz. Yeni bir Task yani Thread veya bizim deyişimizle işçi açmak istersek; Task.Run(() => {Kodlar}) şeklinde açarız.
Kullanabilmek için using System.Threading.Tasks; ile projemize dahil etmeyi unutmuyoruz.
Console.WriteLine("Ana program akışı başladı");

Task.Run(() => 
{
    Console.WriteLine("Yeni bir program akışı açıldı");

    Thread.Sleep(1000); // Hangi Task'daysak onu bekletir. Milisaniye cinsinden

    Console.WriteLine("Yeni açılan program akışı bitti");
});

Thread.Sleep(2000); // Hangi Task'daysak onu bekletir. Milisaniye cinsinden

Console.WriteLine("Ana program akışı bitti");

Kodu çalıştırırsanız eğer, konsolda ilk "Ana program akışı başladı" ve "Yeni bir program akışı açıldı" yazısını görüceksiniz. Bir saniye sonra "Yeni açılan program akışı bitti" yazısı ve bir saniye sonunda da "Ana program akışı bitti" yazısını görüceksiniz. Bunun sebebi, ana program akışı ile yeni program akışımız aynı anda başlıyor. Yeni açtığımız program akışı bir saniye ardından bitiyor. Ana program akışımız ise başladığı andan iki saniye sonra kapanıyor.

Task<Tür> Kullanımı

Bu yapı, Task.Run(); komutunu birer Return'lü fonksiyon olarak kullanma imkanı sağlar.
Öncekinde sadece Task.Run(); yazardık ve içindekini yapar kapatırdı kendi. Şimdi ise onu bir değişkene atıyoruz. Değişkenimizin türü Task<Tür> olucak ve TaskIsmi.Result ile sonucu alabiliriz.
 Console.WriteLine("Program başladı");

 Task<string> sonuc = Task.Run(() =>
 {
     Console.WriteLine("İşlemler yapılıyor...");

     Thread.Sleep(2000);

     Console.WriteLine("İşlemler tamalandı ana program akışında gösteriliyor...");

     return "İşlem sorunsuz bir şekilde tamamlandı";
 });

 Console.WriteLine("İşlemlerin sonucu: " + sonuc.Result);

 Console.WriteLine("Program bitti");

Program çalıştırılsa, yine aynı ana ile yeni program akışları aynı anda başlıyor. Yeni program akışı iki saniye bekliyor ve ardından "İşlem sorunsuz bir şekilde tamamlandı" sonucunu veriyor. Ana program akışınında sonuc.Result ile "İşlem sorunsuz bir şekilde tamamlandı" yazdırabiliyoruz. Fark ettiyseniz Main fonksiyonunda herhangi bir Thread.Sleep() komutu yok. Bunun sebebi; Yeni açılan Task'den gelecek olan bir sonucu beklemesidir. Buradan anlamamız gerek konu eğer bir Task, başka bir Task'den veri istiyor ise Task'lar birbirlerini beklerler.

Async/Await

Asenkron programlama yapmamızı sağlayan araçlardır. Async ile Await kullanımını açarız., Await ile asenkron programı asenkron yaparız.
Geldik asıl asenkron programalamya. Uygulaması için ilk örneğimizi biraz daha geliştirelim.
static async Task YeniProgramAkisi()
{
    Console.WriteLine("Yeni bir program akışı açıldı");

    Thread.Sleep(1000);

    Console.WriteLine("Yeni açılan program akışı bitti");
}

public static async Task Main(string[] args)
{
    Console.WriteLine("Ana program akışı başladı");

    await YeniProgramAkisi();

    Console.WriteLine("Ana program akışı bitti");
}

Yaptığmız işlem, ana ve yeni program akışları başlar. Yeni program akışında işlemler devam eder, o sırada ana program akışıda yeni açılan program akışını bekler o biter bitmez kendisini devam etirir.
