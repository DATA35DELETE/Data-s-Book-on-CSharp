Obje Yönelimli Programlama (Object Oriented Programming)

Obje yönelimli programlama, sınıflar ve dört büyük ayağı olan kalıtım, kapsülleme, çok biçimlilik ve soyutlama gibi yeteneklere programlar veya programlama dillerine OOP yani obje yönelimli programlama denir.
Çoğu durumda avantajlıdır. Çoğu durumda kod tekrarını önliyerek programcıya kolaylık sağlar. Örneğin kalıtım, kalıtılan sınıfın özelliklerini kalıtılanan sınıfa aktarırak kod tekrarlarını önler. Kapsülleme ile değişkenlerin korunasını ve ön koşullar eklenerek güvenliği sağlanır...

Kalıtım

Kalıtım, bir sınıfın özelliklerini diğer bir sınıfa aktarmamızı sağlayan bir özelliktir. Bir nesneyi kalıtmak için "SinifIsmi : KalıtılanSinif" şeklinde yazılır. Bir sınıfın özellikleri ise; fonksiyonları ve değişkenleridir.
Şimdi bir "Canli" sınıfımı oluşturalım. Ardından canlıların ortak özelliklerinden olan beslenme, büyüme ve kareket eylemlerini ekliyelim. Ardından "Kedi" ve "Kopek" isimli iki sınıf oluşturalım ve "Canli" sınıfından kalıtalım.
class Canli // Canlı sınıfımız
{
    // Canlıların ortak özelliklerinden üç tane örnek
    public void Beslenme()
    {
        Console.WriteLine("Canlı beslendi");
    }

    public void Hareket()
    {
        Console.WriteLine("Canlı hareket etti");
    }

    public void Buyume()
    {
        Console.WriteLine("Canlı büyüdü");
    }
}

class Kedi : Canli
{

}

class Kopek : Canli
{

}

internal class Program
{
    static void Main(string[] args) // Ana fonksiyon
    {
        Kedi kedi = new Kedi(); // Kedi canlımzı

        Console.WriteLine("Kedi:");
        kedi.Beslenme(); // Canlı sınıfından gelen özellikler
        kedi.Hareket();  // Canlı sınıfından gelen özellikler
        kedi.Buyume();  // Canlı sınıfından gelen özellikler

        Console.WriteLine("");

        Kopek kopek = new Kopek(); // Köpek canlımız

        Console.WriteLine("Köpek:");
        kedi.Beslenme();  // Canlı sınıfından gelen özellikler
        kedi.Hareket();  // Canlı sınıfından gelen özellikler
        kedi.Buyume();  // Canlı sınıfından gelen özellikler
    }
}

Kapsülleme

Kapsülleme, bir değişkene atama ve çekme işlemlerene ön koşul tanımlamaktır.
Kapsülleme get ve set bloklarıyla gerçekleşir. Get, değişkeni çekme işleminde çağrılan blok; Set ise değişkene atama işleminde cağrılan kod bloğudur. Böylelikle ön koşul elde ederibiliriz. Kapsülleme yaparken, kapsüllediğimiz değişkeni Private olarak ayarlarız. Yoksa kapsüllemenin bir espirisi kalmaz. Ardından public DeğişkenTürü DeğişkenIsmi{...} şeklinde kapsüllememizi yaparız.
Şimdi biliyorsunuz ki yaş negatif sayılar olamazlar. İşte şimdi bu durumu kapsülleme ile uygulayalım. Kullanıcıdan yaşını istiyelim ve eğer yaşı sıfırdan küçük yani negatif girerse uyarı alsın.
class Kullanici // Canlı sınıfımız
{
    private int yas;

    public int Yas
    {
        get // Genelde get pek fazla kullanılmaz ve böyle bırakılır
        {
            return yas;
        }

        set
        {
            // Eşittir ile atanmak istenen değer, bu kısımda "value" isimli bir değişkene atanır

            if (value < 0)
            {
                Console.WriteLine("Yaşınız negatif olamaz!");
            }
            else
            {
                yas = value;

                Console.WriteLine($"Yaşınız: {yas}");
            }
        }
    }
}

internal class Program
{
    static void Main(string[] args) // Ana fonksiyon
    {
        while (true) // Sonsuz döngü olsun ki program kapanmasın
        {
            Kullanici User = new Kullanici(); // Nesnemiz

            Console.Write("Yaşınızı giriniz: ");
            string YasString = Console.ReadLine();

            User.Yas = Convert.ToInt32(YasString); // Kullanıcının girdiği yaşı gönder
        }
    }
}

Çok Biçimlilik

Çok biçimlilik, metodları veya değişkenleri ezmek veya gizlemek için kullanılan bir OOP prensibidir.
Çok biçimlilik için Vitrual ve Override anahtar kelimeleri kullanılır. Vitrual, çok biçimlilik ve Override ile üzerine yazmak yani metodu veya değişkeni ezmek için kullanılır. New ile metodu veya değişkeni gizliyerek, özünü kullanılır. Ayrıca çok biçimlilik için kalıtım gereklidir.
Çok biçimliliği Kedi-Köpek-Canlı örneğimiz olan konusunda daha iyi anlatalım. Şimdi önceki örnekte ne oluyorudu?. Kedi de köpek de "Beslenme" metodunu çağırdığımızda "Canlı beslendi" yazısını alırız. Şimdi bunu değiştirelim ve canlıye özel metod oluşturalım.
class Canli // Canlı sınıfımız
{
    // Canlıların ortak özelliklerinden üç tane örnek
    public virtual void Beslenme()
    {
        Console.WriteLine("Canlı beslendi");
    }

    public virtual void Hareket()
    {
        Console.WriteLine("Canlı hareket etti");
    }

    public virtual void Buyume()
    {
        Console.WriteLine("Canlı büyüdü");
    }
}

class Kedi : Canli
{
    public override void Beslenme()
    {
        Console.WriteLine("Kedi beslendi");
    }
    public new void Hareket()
    {
        Console.WriteLine("Kedi hareket etti");
    }

    public new void Buyume()
    {
        Console.WriteLine("Kedi büyüdü");
    }
}

class Kopek : Canli
{
    public override void Beslenme()
    {
        Console.WriteLine("Köpek beslendi");
    }
    public new void Hareket()
    {
        Console.WriteLine("Köpek hareket etti");
    }

    public new void Buyume()
    {
        Console.WriteLine("Köpek büyüdü");
    }
}

internal class Program
{
    static void Main(string[] args) // Ana fonksiyon
    {
        Kedi kedi1 = new Kedi(); // Klasik nesne oluşturma

        kedi1.Beslenme();
        kedi1.Hareket();
        kedi1.Buyume();

        Console.WriteLine("-----------------");

        Kopek kopek1 = new Kopek(); // Klasik nesne oluşturma

        kopek1.Beslenme();
        kopek1.Hareket();
        kopek1.Buyume();

        Console.WriteLine("-----------------");

        Canli kedi2 = new Kedi(); // Çok biçimlilik ile nesne oluşturma
        // Bunu yapabiliyoruz çünkü "Kedi" sınıfımızın içinde "Canli" sınıfımızda yer almakta

        kedi2.Beslenme(); // Metod ezildiği için konsolda "Köpek beslendi" yazılıcaktır
        kedi2.Hareket(); // Metod gizlendiği için "Canlı hareket etti" yazılıcaktır
        kedi2.Buyume();  // Metod gizlendiği için "Canlı hareket etti" yazılıcaktır

        Console.WriteLine("-----------------");

        Canli kopek2 = new Kopek(); // Çok biçimlilik ile nesne oluşturma

        kopek2.Beslenme(); // Metod ezildiği için konsolda "Kedi beslendi" yazılıcaktır
        kopek2.Hareket();  // Metod gizlendiği için "Canlı hareket etti" yazılıcaktır
        kopek2.Buyume();  // Metod gizlendiği için "Canlı hareket etti" yazılıcaktır
    }
}

Soyutlama

Soyutlama, türetilen sınıftaki özellikleri, türetiline zorunlu kıldıran sınıf türüdür.
Normalde kalıtımda biz kalıtılan özellikler diğer kalıtılanlara aktarılırdı. Soyutlama ise yine kalıtım kullanılır ve soyut sınıftaki özellikleri, diğer kalıtılanlara zorunlu hale getirir ve özelliklerine Override ile üzerine yazdırılır.
Daha iyi anlamız için kalıtım örneğimiziyle aynı işlevi gören fakat soyutlama kullanan bir örnek yazalım.
abstract class Canli // Canlı sınıfımız
{
    public abstract void Beslenme(); // Soyut özelliklerin gövdesi yani içeriği olmaz. Çünkü Override ile ezilcekleri için mantıksız olur
    public abstract void Hareket(); // Soyut özelliklerin gövdesi yani içeriği olmaz. Çünkü Override ile ezilcekleri için mantıksız olur
    public abstract void Buyume(); // Soyut özelliklerin gövdesi yani içeriği olmaz. Çünkü Override ile ezilcekleri için mantıksız olur

}

// Eğer soyut sınıftaki özellikler, sınıflarda Override edilmesse, hata alınıcaktır
